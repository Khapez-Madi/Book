1. База данных MS ACCESS

Так как она является приложением WINDOWS, то может использовать обмен данными с другими приложениями WINDOWS: Word, Excel. В нее можно внедрять также различные рисунки, диаграммы, документы. И наоборот: данные из ACCESS можно использовать в Word, Excel. Кроме того можно импортировать данные из некоторых других СУБД.

Кроме таблиц В СУБД ACCESS можно создавать следующие объекты: запросы, формы, отчеты, макросы, модули. Все эти объекты БД вместе с таблицами хранятся в одном файле с расширением .mdb. Большинство приложений можно создавать, не использую язык программирования, но если необходимо создать достаточно сложное приложение, то используется язык Visial Basic.

Основными компонентами БД являются таблицы. Они состоят из строк (записей) и столбцов (полей).При создании таблицы надо создать ее структуру. Структура таблицы включает в себя название и тип полей и их размеры. Каждому полю присваивается уникальное имя. Задаются ключевые поля. Поля должны быть независимы друг от друга. У таблиц существует 2 режима: режим конструктора и режим таблицы. Структура таблицы создается в режиме конструктора.

2. Типы данных

1.Текстовый. Содержит текст или числа, не требующие проведения расчетов (например, номер телефона). Размер поля может включать до 255 символов, его можно изменять.

2. Числовой. Содержит данные различных форматов, которые используются для проведения расчетов.

3. Дата/время

4.Денежный.

5. MEMO. Используется для хранения комментариев. Размер может достигать 65 535 символов.

7. Счетчик. Специальное числовое поле, в котором автоматически присваивается уникальный номер записи. Этот номер вручную изменить нельзя. По типу чисел является длинным целым.

8. Логический (Да/Нет).

9. Гиперссылка. Адрес в Интернете.

10. Поле объекта. Это могут быть электронные таблицы, документы Word, картинки и т.д., связанные или внедренные.

3. Режим таблицы

В этом режиме на экране отображается сама таблица. Вообще работать в режиме таблицы не рекомендуется, для этого есть другие объекты, а в таблице данные должны только храниться, так и делают пользователи серьезных баз данных.

Но если это только ваша таблица, что возможно делать в режиме таблицы?

1. Вручную (хотя это делать не рекомендуется)

а) заполнять данными, хотя это не всегда возможно, и не всегда удобно

б) редактировать данные (тоже не всегда возможно)

в)добавлять данные.

2. Автоматически

а) сортировать данные, т.е. изменять порядок строк по какому-либо полю (текстовые – по алфавиту, числовые – по возрастанию или убыванию). Можно сортировать по нескольким соседним столбцам, при этом сортировка начинается с крайнего левого выделенного столбца.

б) производить поиск данных. Можно производить отбор при помощи фильтра.

в) заменять какие-либо символы на другие. Например, все точки заменить на запятые.

Можно поменять местами столбцы, удалить столбцы. Можно напечатать таблицу.

В режиме таблицы нет возможностей:

1. Делать вычисления.

2. Нельзя вставить строку в любом месте таблицы. Строки в таблице размещаются в порядке заполнения. Можно сделать сортировку по номерам, можно использовать формы, запросы.

4. Формы

Являются специально оформленными бланками (шаблонами) для более удобной работы с БД. Позволяют работать сразу с несколькими объектами (таблицами, запросами).

1.В форме может располагаться много полей на одном экране, это удобно, когда в таблице много полей и они не умещаются на одном экране. Каждое поле можно разместить в любом месте экрана.

2. Можно отображать не все поля, а только нужные, причем , из разных таблиц, запросов.

3. Можно помещать вычисляемые поля.

4. Можно создавать различные кнопки для выполнения макросов, для выполнения процедур, написанных на языке программирования.

Используются для:

1.Ввода данных;

2. Поиска, редактирования

3. Управления ходом выполнения приложения

4. Вывода сообщений

5. Печати информации.

Все то, что отражается на экране не сохраняется в виде данных. Данные берутся из таблиц. Сохраняются только структуры форм. То же относится и к запросам, и к отчетам.

5. Запросы

Это основное средство манипуляции с данными. Имеют много общего по возможностям с формами. По действиям с данными возможностей больше, но нет таких средств оформления. Имеют большие возможности для выбора, сортировки и вычислений. Можно использовать сразу несколько таблиц и других запросов.

Виды запросов

 Основной вид – запросы на выборку.

Порядок выполнения:

-выбираются нужные таблицы и другие запросы.

- задаются из всех выбранных объектов только нужные поля

- задаются условия отбора и сортировка, если это необходимо.

К полученной на экране таблице можно добавить дополнительные столбцы (поля), которые будут содержать результаты вычислений.

Записи можно сгруппировать и ,например, вычислить сумму, среднее арифметическое, min, max значений выбранного поля. Такие запросы называются итоговыми.

Повторю, что все что мы видим на экране, на диске не сохраняется.

 Запросы на создание таблицы. Из созданного запроса на выборку создается таблица.

 Запросы модификации данных. Изменяют данные в таблицах. Применяют, когда надо изменить данные сразу во многих записях таблицы. Существует 3 вида запросов на модификацию:

- обновление, т.е. изменение данных

- добавление – автоматически добавляется заданное количество записей

- удаление записей.

Результаты запросов на модификацию записываются в таблицу. Прежде чем выполнить этот вид запросов, рекомендуется создать копию таблицы, в которой хотите сделать изменения, т.к. можно испортить таблицу и восстановить будет нельзя, т.к. все изменения сразу же записываются на диск (в отличие от WORD и EXCEL).

 Перекрестные запросы. Это запросы особого вида. Например: в таблице «Преподаватели» надо узнать средний стаж работы ассистентов, доцентов и профессоров на разных кафедрах.

6. Отчеты

Используются для формирования, просмотра и распечатки документов. Можно вывести результаты различных расчетов, статистических сравнений, сумм. Можно создать различное оформление и использованием рисунков, диаграмм. Обычно для формирования отчета сначала создается нужный запрос.

Макросы.

Различные действия автоматически записываются в виде программы. Затем их можно выполнить.

Этапы проектирования БД

1.Продумать какие надо создать таблицы и какие в них будут поля, какие будут связи и по каким полям. Какие поля надо добавить для создания связей.

2.В режиме конструктора создать структуру таблиц.

3.Создать связи. Связи можно устанавливать только с полями одинакового типа.

4.Создать формы для заполнения таблиц данными.

5.Заполнить таблицы с использованием форм.
